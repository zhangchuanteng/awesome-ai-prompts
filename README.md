## ✍️ 通用写作

### 1. 文章撰写专家

你是一位资深的[领域]专家，请帮我撰写一篇关于[主题]的文章。要求：
1. 标题吸引眼球，包含关键词
2. 开头用一个引人入胜的故事或数据切入
3. 正文分 3-5 个要点，每个要点配案例说明
4. 结尾给出行动建议
5. 全文控制在 [字数] 字左右
6. 语言风格：[专业/轻松/学术/口语化]

### 2. 小红书爆款文案

请帮我写一篇小红书风格的笔记，主题是 [主题]。要求：
1. 标题用 emoji 开头，包含数字和痛点词（如"后悔没早知道"）
2. 开头 3 秒抓住注意力（提问/反转/震惊）
3. 正文用 emoji 分段，每段不超过 3 行
4. 口语化、有亲和力、像朋友聊天
5. 结尾引导互动（收藏/评论/关注）
6. 添加 5-8 个相关话题标签
7. 总字数 500-800 字

### 3. 公众号深度文章

请写一篇微信公众号文章，主题：[主题]。要求：
1. 标题：疑问句或数字型（如"为什么90%的人不知道…"）
2. 开头用故事/案例/数据引入
3. 正文 3-5 个小标题，层层递进
4. 每个观点配具体案例或数据支撑
5. 结尾引发思考，引导点赞转发
6. 语言风格：有深度但不学术，像资深朋友分享
7. 总字数 2000-3000 字

### 4. 深度文章改写

请将以下文章改写为更通俗易懂的版本，同时保持核心观点不变。

原文：[粘贴原文]

要求：
1. 将专业术语替换为日常表达
2. 添加生活化的类比和例子
3. 段落之间增加过渡句
4. 保持信息完整性
5. 目标读者：[描述目标读者]

### 5. 文案润色大师

请帮我润色以下文字，使其更加优美流畅：

[粘贴文字]

要求：
1. 保持原意不变
2. 提升文字的画面感和节奏感
3. 去除冗余表达
4. 增加恰当的修辞（比喻、排比等）
5. 控制句子长度，长短句交替

### 6. 标题生成器

请为主题 [主题] 生成 10 个不同风格的标题：

1. 数字型（"7个方法..."）
2. 疑问型（"为什么..."）
3. 对比型（"X vs Y"）
4. 揭秘型（"不为人知的..."）
5. 情感型（"后悔没早知道..."）
6. 权威型（"专家都在用的..."）
7. 紧迫型（"再不...就晚了"）
8. 故事型（"我如何从...到..."）
9. 清单型（"这10件事..."）
10. 反常识型（"你以为的...其实是..."）

## 💻 编程开发

### 7. 代码审查专家

请作为资深代码审查专家，审查以下代码：

```[语言]
[粘贴代码]
```

从以下维度审查：
1. **正确性**：逻辑是否正确，边界条件是否处理
2. **性能**：是否有性能瓶颈，算法复杂度是否合理
3. **安全性**：SQL注入、XSS、CSRF 等漏洞检查
4. **可读性**：命名规范、注释充分、代码结构清晰
5. **设计模式**：是否遵循 SOLID 原则
6. **改进建议**：给出具体的优化代码

### 8. SQL 优化大师

请分析并优化以下 SQL 语句：

```sql
[粘贴SQL]
```

表结构：[粘贴表结构]

分析：
1. 执行计划分析（是否有全表扫描）
2. 索引优化建议（哪些列需要索引）
3. 查询重写（用 JOIN 替代子查询等）
4. 潜在性能问题
5. 给出优化后的完整 SQL

### 9. API 设计顾问

业务场景：[描述场景]

输出：
1. 遵循 RESTful 最佳实践的 URL 设计
2. 统一响应格式（成功/失败/分页）
3. 请求/响应示例
4. 错误码设计
5. 认证方案
6. OpenAPI 3.0 格式的接口定义

### 10. 重构大师

请帮我重构以下代码：

```[语言]
[粘贴代码]
```

要求：
1. 遵循 SOLID 原则

3. 适当使用设计模式
4. 添加必要的注释和文档
5. 保持功能不变
6. 解释每一步重构的原因和收益

### 11. 技术方案设计

请帮我设计 [系统名称] 的技术方案：

业务需求：[描述需求]

请包含：

6. 容灾备份方案
7. 监控告警方案
8. 部署架构

### 12. Bug 定位助手

请帮我定位以下 Bug：

**现象**：[描述 Bug 表现]

### 13. Git Commit Message 生成器

请根据以下代码变更生成规范的 Git commit message：

- 背景：[详细背景]
- 性格：[性格特点]
- 知识范围：[专业领域]
- 说话风格：[风格描述]

规则：
1. 始终保持角色设定
2. 如果问题超出你的知识范围，诚实说明
3. 不要打破第四面墙

请用这个角色的身份和我说话。

### 40. 模拟面试官

你是一位 [公司名] 的 [职位] 面试官。请对我进行模拟面试。

岗位：[岗位名称]
公司类型：[互联网/金融/创业公司等]
面试轮次：[技术面/HR面/总监面]

规则：
1. 每次问一个问题
2. 等我回答后给出点评和改进建议
3. 然后问下一个问题
4. 面试结束后给出综合评价和建议
5. 包含 1-2 个压力面试问题

### 41. 辩论对手

请扮演一个辩论对手，与我讨论以下话题：

话题：[话题]
我的立场：[你的观点]

要求：
1. 站在对立面提出反驳
2. 每个反驳都要有论据支撑
3. 逻辑严密，不人身攻击
4. 最后总结双方观点

### 42. 创意头脑风暴伙伴

目标：[要达成的目标]
已有想法：[你已经想到的]

规则：
1. 不批评任何想法（先发散后收敛）
2. 每次提出 5 个想法
