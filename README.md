## 通用写作

### 文章撰写专家

你是一位资深的[领域]专家，请帮我撰写一篇关于[主题]的文章。要求：
1. 标题吸引眼球，包含关键词
2. 开头用一个引人入胜的故事或数据切入
3. 正文分3-5个要点，每个要点配案例说明
4. 结尾给出行动建议
5. 全文控制在[字数]字左右
6. 语言风格：[专业/轻松/学术/口语化]

### 小红书爆款文案

请帮我写一篇小红书风格的笔记，主题是[主题]。要求：
1. 标题用emoji开头，包含数字和痛点词
2. 开头3秒抓住注意力（提问/反转/震惊）
3. 正文用emoji分段，每段不超过3行
4. 口语化、有亲和力、像朋友聊天
5. 结尾引导互动（收藏/评论/关注）
6. 添加5-8个相关话题标签

### 深度文章改写

请将以下文章改写为更通俗易懂的版本，同时保持核心观点不变。

原文：[粘贴原文]

要求：
1. 将专业术语替换为日常表达
2. 添加生活化的类比和例子
3. 段落之间增加过渡句
4. 保持信息完整性
5. 目标读者：[描述目标读者]

### 周报生成器

请根据以下工作内容帮我写一份周报：

工作内容：
[列出本周完成的事项]

要求：
1. 格式清晰，分为"本周完成"、"进行中"、"下周计划"
2. 用数据量化成果（如有）
3. 突出关键成果和价值
4. 语言简洁专业
5. 控制在300字以内

### 邮件撰写

请帮我写一封[类型]邮件，场景如下：

收件人：[职位/关系]
目的：[核心诉求]
背景：[简要背景]

要求：
1. 主题行简洁有力
2. 开头问候得体
3. 正文逻辑清晰，重点前置
4. 结尾明确下一步行动
5. 语气：[正式/半正式/友好]

## 编程开发

### 代码审查专家

请作为一名资深代码审查专家，审查以下代码：

```[语言]
[粘贴代码]
```

从以下维度进行审查：
1. **正确性**：逻辑是否正确，边界条件是否处理
2. **性能**：是否有性能瓶颈，算法复杂度是否合理
3. **安全性**：是否存在安全漏洞（SQL注入、XSS等）
4. **可读性**：命名是否规范，注释是否充分
5. **设计模式**：是否遵循SOLID原则
6. **改进建议**：给出具体的优化代码

### SQL 优化大师

请分析并优化以下SQL语句：

```sql
[粘贴SQL]
```

表结构：
[粘贴表结构]

请从以下方面分析：
1. 执行计划分析
2. 索引优化建议
3. 查询重写（如有更优写法）
4. 潜在的性能问题
5. 给出优化后的完整SQL

### API 设计顾问

请帮我设计一个RESTful API，需求如下：

业务场景：[描述场景]
核心实体：[列出实体]
主要操作：[列出CRUD操作]

要求：
1. 遵循RESTful最佳实践
2. URL命名规范
3. 统一响应格式
4. 包含分页、过滤、排序设计
5. 错误码设计
6. 给出OpenAPI/Swagger格式的接口定义

### 重构大师

请帮我重构以下代码，使其更加优雅：

```[语言]
[粘贴代码]
```

要求：
1. 遵循SOLID原则
2. 消除代码异味（重复、过长方法、魔法数字等）
3. 适当使用设计模式
4. 添加必要的注释
5. 保持功能不变
6. 解释每一步重构的原因

### 技术方案设计

请帮我设计一个[系统名称]的技术方案：

业务需求：[描述需求]
预期规模：[QPS/用户量/数据量]
技术栈：[列出技术栈]

请包含以下内容：
1. 系统架构图（文字描述）
2. 核心模块设计
3. 数据库设计（ER图描述）
4. 关键流程时序图
5. 性能优化策略
6. 容灾备份方案
7. 监控告警方案

## AI 绘画

### Midjourney 风格提示词

[主体描述], [艺术风格], [光照效果], [镜头角度], [画质修饰]

示例：
A majestic dragon flying over a misty mountain range at sunset, 
fantasy art style, dramatic golden hour lighting, wide-angle shot, 
8K resolution, highly detailed, cinematic composition --ar 16:9 --v 6

### 通用绘画公式

[Subject] + [Medium] + [Style] + [Artist Reference] + [Color] + [Lighting] + [Composition] + [Quality Tags]

中文模板：
[主体]，[媒介]，[风格]，[参考艺术家]，[色调]，[光照]，[构图]，[质量词]

示例：
一只橘猫坐在书堆上阅读，水彩插画风格，吉卜力风，暖色调，柔和自然光，
俯视构图，高质量，细节丰富

### Logo 设计提示词

请为[品牌名]设计一个Logo，品牌定位：[定位描述]

要求：
1. 风格：[极简/科技/复古/可爱/高端]
2. 主色调：[颜色]
3. 包含元素：[图标/字母/图形]
4. 用途：[APP图标/网站/名片]
5. 给出3个不同方案
6. 每个方案附带Midjourney/SD提示词

## 营销文案

### 品牌故事撰写

请为[品牌名]撰写一个品牌故事：

品牌信息：
- 行业：[行业]
- 产品/服务：[描述]
- 目标用户：[用户画像]
- 核心价值：[价值主张]
- 创始故事：[如有]

要求：
1. 用"英雄之旅"叙事结构
2. 情感共鸣，引发认同
3. 突出差异化
4. 控制在500字以内
5. 适合用于官网"关于我们"页面

### 产品卖点提炼

请为以下产品提炼核心卖点：

产品名称：[名称]
产品描述：[描述]
目标用户：[用户画像]
竞品分析：[竞品信息]

要求：
1. 使用FAB法则（Feature-Advantage-Benefit）
2. 提炼3-5个核心卖点
3. 每个卖点用一句话概括
4. 配套一个使用场景描述
5. 给出对应的广告语

### 社交媒体内容规划

请为[品牌/个人]制定一周的社交媒体内容计划：

平台：[小红书/抖音/公众号/微博]
行业：[行业]
目标：[涨粉/引流/品牌曝光]
受众：[目标人群]

要求：
1. 每天1条内容主题
2. 包含内容类型（图文/视频/互动）
3. 标题建议
4. 话题标签
5. 发布时间建议
6. 周末安排互动性内容

## 学习教育

### 费曼学习法

请用费曼学习法帮我理解[概念/知识点]：

要求：
1. 用最简单的语言解释（假设听众是10岁小孩）
2. 使用生活中的类比
3. 举一个具体的应用实例
4. 指出常见的误解
5. 提出3个检验理解程度的问题

### 知识图谱构建

请为[学科/领域]构建一个学习路线图：

我的背景：[当前水平]
学习目标：[期望达到的水平]
可用时间：[每天/每周可投入时间]

请包含：
1. 核心知识模块
2. 学习顺序和依赖关系
3. 每个模块的学习资源推荐
4. 预计学习时间
5. 阶段性检验方法
6. 常见学习陷阱和建议

### 读书笔记模板

请帮我整理以下书籍的读书笔记：

书名：[书名]
作者：[作者]

请按照以下框架整理：
1. 一句话总结
2. 核心观点（3-5个）
3. 金句摘录
4. 个人感悟
5. 行动清单（读完后可以立即做的3件事）
6. 相关书籍推荐

## 效率工具

### 会议纪要生成

请根据以下会议录音/笔记生成会议纪要：

会议内容：
[粘贴会议记录或要点]

要求：
1. 包含：会议主题、时间、参会人
2. 讨论要点摘要
3. 决议事项（明确责任人和截止日期）
4. 待办事项清单
5. 遗留问题
6. 格式简洁，便于发送

### 简历优化

请帮我优化以下简历：

目标岗位：[岗位名称]
当前简历内容：
[粘贴简历]

请从以下方面优化：
1. 针对目标岗位调整关键词
2. 用量化的数据描述成果
3. 使用STAR法则改写项目经历
4. 优化排版和格式
5. 突出与岗位最匹配的技能
6. 给出修改前后的对比

### PPT 大纲生成

请为以下主题生成一个PPT大纲：

主题：[主题]
时长：[演讲时长]
听众：[听众画像]
目的：[汇报/培训/路演]

要求：
1. 包含封面、目录、正文、总结、Q&A
2. 每页的核心观点
3. 建议的视觉元素（图表/图片）
4. 演讲者备注要点
5. 控制在[页数]页以内

## Prompt 技巧

### 结构化 Prompt 模板

```
# Role: [角色名称]

## Profile
- Author: [作者]
- Version: 1.0
- Language: 中文
- Description: [角色描述]

## Skills
- [技能1]
- [技能2]
- [技能3]

## Rules
1. [规则1]
2. [规则2]
3. [规则3]

## Workflow
1. [第一步]
2. [第二步]
3. [第三步]

## Initialization
你好，我是[角色]，请告诉我你需要什么帮助？
```

### Few-Shot 示例模板

请按照以下示例的格式和风格，完成新的任务：

示例1：
输入：[输入内容1]
输出：[输出内容1]

示例2：
输入：[输入内容2]
输出：[输出内容2]

现在请处理：
输入：[新的输入内容]
输出：

### Chain of Thought 推理模板

请一步一步思考[问题]。

要求：
1. 首先，明确问题的核心
2. 列出所有可能的影响因素
3. 对每个因素进行分析
4. 综合分析得出结论
5. 给出最终建议

请在每个步骤前标注"Step X："，并详细展示你的思考过程。

## 贡献指南

欢迎贡献！请遵循以下规范：

1. **格式统一**：每个 Prompt 包含标题、描述、完整提示词
2. **分类明确**：将 Prompt 归入合适的分类
3. **质量优先**：确保 Prompt 实用、有效
4. **注明来源**：如非原创请注明出处

### 提交方式

1. Fork 本仓库
2. 创建分支：`git checkout -b add-new-prompt`
3. 提交修改：`git commit -m "Add: [prompt名称]"`
4. 推送分支：`git push origin add-new-prompt`
5. 创建 Pull Request
